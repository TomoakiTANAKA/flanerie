# Flanerie

教育用3D街探索ゲーム

## 🎯 プロジェクト概要

### コンセプト
歴史的な街並みを3D空間で探索し、NPCとの会話や建物の詳細情報を通じて学習する教育用インタラクティブコンテンツ

### 主な特徴
- **神視点操作**: プレイヤーキャラクターは存在せず、カメラのみ移動
- **コンパクトなマップ**: 一つの街区・商店街・城下町レベルの小規模マップ
- **情報収集型探索**: Hidden ObjectやPoint & Click風の学習体験

---

## 🎮 ゲームデザイン

### 操作システム
- **視点**: RTSライクな俯瞰視点（Coffee Inc 2のような規模感）
- **カメラ操作**: 
  - 十字キー/WASD: 平行移動
  - マウスドラッグ: 視点回転
  - ホイール: ズーム
- **インタラクション**: クリック/タップでオブジェクト詳細表示・NPC会話

### 学習要素
- **NPCとの会話**: 街の住民から歴史や文化の情報収集
- **建物解説**: 各建築物をクリックして詳細情報表示
- **情報収集ミッション**: 「どういう人が住んでいるか？」「どういう問題があるか？」等のテーマ探索

---

## 🏛️ コンテンツテーマ例

### 歴史的街並み
- **江戸時代の東京**: 江戸城、長屋、商家、寺社
- **中世ヨーロッパ**: 城、教会、市場、職人街
- **明治時代の街**: 洋風建築と和風建築の混在

### 現代応用
- **現代の街**: 職業体験、社会の仕組み学習
- **テーマパーク**: 特定分野の体験学習
- **博物館/水族館**: 展示見学と解説

## 💻 技術仕様

### 基盤技術
- **フロントエンド**: Next.js + Three.js
- **3D表現**: 基本形状の組み合わせ → 必要に応じてGLTF導入
- **プラットフォーム**: Webブラウザ（学校環境での利用を重視）

### 開発者スキルベース
- **既存スキル**: Phaser3, Next.js, Unity, Flutter
- **学習予定**: Three.js（基本的な3Dシーン構築）

### アーキテクチャ
```
Next.js アプリケーション
├── Three.js 3Dマップ部分
├── Phaser3 UI・会話システム
└── Google Maps API（将来的な拡張）
```

## 🔧 技術的課題と解決策

### 主な課題
1. **3D表現の品質 vs 開発コスト**
   - **解決策**: 段階的品質向上（基本形状 → GLTF）
   
2. **学校環境での動作保証**
   - **解決策**: 軽量化・最適化、多様なデバイス対応
   
3. **コンテンツ制作の効率化**
   - **解決策**: テンプレート化、モジュール設計

### リスク対策
- **技術リスク**: Three.js学習、段階的実装
- **市場リスク**: 教育現場でのテスト・フィードバック重視
- **競合リスク**: 差別化機能（Google Maps連携、時代変遷）
